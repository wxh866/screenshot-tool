## 二、问题清单

### 🔴 缺陷 #1：多边形工具无法闭合（真实 Bug，P0）

- **文件**: `engines/annotation/polygon.py`、`controllers/editor_controller.py`、`views/EditorView.qml`
- **复现**: 单元测试 `test_polygon_tool` 报 `AttributeError: 'EditorController' has no attribute 'finishPolygon'`，多边形双击闭合路径完全缺失
- **根因**: 编辑器从未提供多边形闭合入口；QML 端用 `polygonDoubleTimer`（Timer 模拟双击）方案有缺陷，Python 侧无对应 Slot 接收双击事件，导致多边形只能添加顶点、无法结束绘制
- **参考**: Flameshot 多边形以双击/Enter 结束，`onDoubleClick` 仅用于闭合、不重复追加顶点
- **修复**:
  1. `polygon.py`：将 `onDoubleClick` 重命名为 `onMouseDoubleClick`，移除重复追加顶点逻辑，双击仅设 `_is_closed=True` 并 `finishAnnotation()`
  2. `editor_controller.py`：新增 `@Slot(int,int) finishPolygon(x,y)`，校验当前工具为 POLYGON 后去重连续重复顶点并调用 `onMouseDoubleClick`
  3. `EditorView.qml`：移除有缺陷的 `polygonDoubleTimer`，改用 Qt 原生 `onDoubleClicked` → `EditorController.finishPolygon(mouse.x, mouse.y)`
- **验证**: TC-12 通过，多边形可正常闭合

### 🔴 缺陷 #2：智能选区 GrabCut 为空壳（真实 Bug，P0）

- **文件**: `engines/annotation/smart_select.py`、`controllers/editor_controller.py`
- **复现**: 单元测试 `test_smart_select_tool` 发现撤销备份栈 = 0，智能选区未对图像产生任何修改
- **根因**: `onMouseRelease` 仅设置 `properties["algorithm"]="grabcut"`，从未调用 `cv2.grabCut`；`SMART_SELECT` 也未纳入破坏性工具集合，导致选区操作被当作非破坏性叠加，实际不执行分割
- **参考**: JamTools 智能选区使用 OpenCV `grabCut` 做前景分割，选区外做模糊/去色处理形成人像模式效果
- **修复**:
  1. `editor_controller.py`：`_DESTRUCTIVE_TOOLS` 增加 `ToolType.SMART_SELECT`；`_applyDestructiveTool` 增加分支路由到 `_applySmartSelect`
  2. 新增 `_applySmartSelect(img, ann)`：计算选区矩形 → `cv2.grabCut` 5 次迭代 → 取前景掩码 → 选区外灰度化 + 高斯模糊 → 合成输出；失败时回退为选区外轻微模糊
- **验证**: TC-11 通过，备份栈 ≥ 1，选区外背景去色+模糊生效

### 🟠 缺陷 #3：QRect 闭区间导致坐标 +1 偏移（真实 Bug，P1）

- **影响文件**: `engines/annotation/rect.py`、`circle.py`、`highlight.py`、`mosaic.py`、`smart_select.py`
- **复现**: 单元测试 `test_rect_tool` 期望存储坐标 `[200,150]`，实际得到 `[201,151]`
- **根因**: 工具用 `QRect(QPoint, QPoint)` 构造（`width = x2 - x1 + 1` 闭区间），再用 `rect.x()+rect.width()` 取右下角时多加了 1，导致存储/渲染的右下角比光标实际落点偏右下 1 像素
- **修复**: 统一改用 `rect.right()` / `rect.bottom()` 取包含端点坐标
  ```python
  # 旧（偏 +1）
  self.points = [(rect.x(), rect.y()), (rect.x()+rect.width(), rect.y()+rect.height())]
  # 新（精确）
  self.points = [(rect.x(), rect.y()), (rect.right(), rect.bottom())]
  ```
- **验证**: 直接驱动工具得 `[200,150]`，控制器路径得 `[201,151]`，确认为 QRect 行为而非测试污染；修正后 TC-01/02/06/07/11 全部正确
